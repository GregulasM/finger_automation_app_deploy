# Finger_automation_app
Finger - приложение для автоматизации, основанное на No-Code идеи. Карточки с сервисами, разные настройки и взаимодействие между ними.

## Страницы приложения

### Главная страница (`/`)
- Лендинг с описанием возможностей приложения
- Кнопки навигации: открыть редактор workflow, перейти к дашборду, войти в систему
- Информационные блоки о функциональности: webhook-триггеры, cron-расписания, JSON-first workflow
- Описание преимуществ: от триггера к действию, надежное выполнение, логи для аудита

### Страница входа (`/auth/login`)
- Форма аутентификации с полями email и пароль
- Валидация данных (email формат, минимальная длина пароля)
- Обработка ошибок входа с отображением сообщений
- Автоматический редирект на редактор после успешного входа
- Ссылка на страницу регистрации

### Страница регистрации (`/auth/register`)
- Форма регистрации с полями: имя, email, пароль
- Валидация имени (кириллица, латиница, 2-80 символов)
- Валидация email и пароля
- Обработка ошибок регистрации
- Автоматический редирект на редактор после успешной регистрации
- Ссылка на страницу входа

### Дашборд workflow (`/workflows`)
- Список всех workflow пользователя с информацией о статусе и типе триггера
- Детальный просмотр выбранного workflow:
  - Общая информация (название, статус, тип триггера)
  - Статистика выполнения (общее количество, процент успеха, средняя длительность)
  - История последних выполнений
  - Webhook endpoint для workflow с триггером типа WEBHOOK
- Управление workflow: редактирование, удаление
- Адаптивный интерфейс: список/детали на мобильных, split-view на десктопе
- Кнопка создания нового workflow

### Редактор workflow (`/workflows/editor`)
- Визуальный редактор для создания и редактирования workflow
- Настройка названия и статуса активности workflow
- Выбор типа триггера (WEBHOOK, EMAIL, CRON)
- Конфигурация триггера (расписание для CRON, настройки для EMAIL)
- Построение цепочки действий (actions) с настройкой параметров каждого шага
- Сохранение workflow с валидацией данных
- Редактирование существующего workflow по query-параметру `workflowId`
